# Datasheet for Datasets

## Dataset name

**Synthetic mtDNA Dual-Reporter Programmability Datasets**

## Version

Version 1.0

## Motivation

These datasets were created to support computational exploration of mitochondrial programmability for an engineered **mtDNA-like nucleic acid cargo** carrying **mtNanoLuc** and **sfGFP** reporter modules.

The purpose of the datasets is to provide a structured synthetic benchmark for studying how cargo design, delivery strategy, and mitochondrial state may interact across a serial sequence of biological barriers.

## Composition

The project includes two primary datasets:

### 1. Main dataset
**File:** `simulated_mtdna_mtnanoluc_sfgfp_dataset.csv`

This dataset contains one row per virtual mitochondrial programming attempt.

### 2. Longitudinal dataset
**File:** `simulated_mtdna_mtnanoluc_sfgfp_longitudinal.csv`

This dataset contains time-resolved synthetic observations derived from the main dataset across four time points.

## Number of instances

### Main dataset
Approximately 8,000 rows

### Longitudinal dataset
Approximately 14,000 rows

## What each row represents

### Main dataset
Each row represents one synthetic experiment involving:

- one mtDNA-like nucleic acid cargo design
- one cell-state background
- one delivery mode
- one target mode
- one set of computed barrier probabilities
- one set of downstream reporter and mitochondrial outcomes

### Longitudinal dataset
Each row represents one synthetic time-point observation derived from a simulated experiment.

## Data generation process

The datasets were generated algorithmically by a rule-based simulator with probabilistic sampling.

The simulator first samples:

- cell state
- delivery mode
- target mode
- cargo design features
- mitochondrial-state features

It then computes sequential barrier probabilities and final outcomes using mathematical rules designed to reflect a serial mitochondrial-control problem.

The longitudinal dataset is generated by interpolating baseline and post-intervention values across multiple time points.

## Missing data

The synthetic datasets are designed to be complete. Missing values are not expected under standard generation settings.

## Relationships among fields

The data are not independently sampled field by field. Many variables are mathematically coupled.

Examples include:

- `Phi_prog` depends on the product of several barrier probabilities
- reporter outputs depend partly on `Phi_prog` and expression compatibility
- post-treatment mitochondrial outcomes depend on functional-gain calculations
- longitudinal values depend on interpolation between pre- and post-intervention states

## Recommended uses

These datasets are appropriate for:

- machine learning benchmarking on synthetic biological systems
- serial bottleneck analysis
- sensitivity analysis
- feature-importance analysis
- educational demonstrations
- exploratory design ranking

## Non-recommended uses

These datasets should not be used as:

- experimental evidence
- clinical data
- regulatory evidence
- direct proof of mitochondrial gene delivery feasibility
- a substitute for wet-lab validation

## Known limitations

1. The datasets are fully synthetic.
2. The underlying equations are mechanistically inspired but simplified.
3. Real mitochondrial nucleic-acid delivery remains much less established than protein import.
4. Reporter-related variables are conceptual and do not prove actual mitochondrial expression of the exact construct modeled.
5. Strong classification performance on synthetic labels does not guarantee external validity.
